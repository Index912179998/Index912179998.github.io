Java异常简述

目前大多编程语言都提供成熟的异常机制，异常机制能够使程序中的异常处理代码和正常业务代码分离，提高程序的健壮性。增加了异常处理机制后的程序有更好的容错性，更加健壮。
当程序运行出现意外情形时，系统会自动生成一个Exception对象来通知程序，从而把功能实现代码和错误处理代码分离，提高更好的可读性。

异常体系

Java把所有的非正常情况飞卫两种：异常（Exception）和错误（Error），他们都继承Throwable父类。
注：Error错误，一般是指与虚拟机相关的问题，如系统泵快、虚拟机错误、动态链接失败等，这些错误无法恢复或者不可能捕获，将导致程序中断，通常应用程序无法处理这些错误，所以不应用try...catch捕获，也不应用throws抛出。

使用try...catch捕获异常

例子：输入五个数字，判断是否符合规定

import java.util.*;
public class Demo
{
    public static Scanner sc = new Scanner(System.in);
    public static void main(String[] args){
        calculation();
    }
    public static void calculation(){
        boolean pp = true;
        int[] num = new int[5];
        while(pp){
            System.out.println("Please enter five integer number:(Separated by commas)");
            try{
                String input = sc.next();//用字符串接收输入的数字
                input = input.trim();//去掉字符串首尾的空格
                String[] temp = input.split(",");//识别逗号,以逗号分隔输入的数字字符，并存入字符串数组中
                for(int i = 0;i<temp.length;i++){
                    num[i] = Integer.parseInt(temp[i]);//将数字字符转化为整形数字
                    //String s1 = String.valueOf(i);//基本类型 --> String
                }
                pp = false;
            }catch(NumberFormatException e){
                System.out.println("Enter data format error!");
            }catch(ArrayIndexOutOfBoundsException n){
                System.out.println("Enter data amount error!");
            }
        }
        for(int x:num){
            System.out.print(x+"\t");
        }
    }
}

Finally回收资源

因为Java的垃圾回收机制不会回收任何物理资源，垃圾回收机制只能回收堆内存中对象所占用的内存。
为了保证能回收try块中打开的物理资源，异常机制提供了Finally块。不管try块中的代码是否出现异常，也不管哪一个catch块被执行，甚至在try块或catch块中执行了return语句，finally块总会被执行。

public class Test
{
    public static void main(String[] args){
        int a = test();
        System.out.println(a);
    }

    public static int text(){
        try{
            //因为finally块中包含了return语句，所以下面的return语句失去作用
            return 1;
        }finally{
            return 2;
        }
    }
}

使用throws声明抛出异常

当不知道如何处理当前类型的异常时，由上一级调用者处理，例如常见mian方法里出异常，JVM的处理方法是，打印异常的跟踪栈信息，并终止程序运行。
throws声明抛出只能在方法签名中使用，throws可以生命抛出多个异常类，多个异常类之间可以用逗号隔开。

import java.util.*;
public class Demo
{

    }

        int a = 0;
        boolean pp = true;
        while(pp){
            try{
                a = sc.nextInt();
                dist(a);
                pp = false;
            }catch(Exception e){
                System.out.println(e.getMessage());
                System.out.println("请重新输入:");
            }
        }
        switch(a){
            case 1:enter();
            case 2:login();
            case 3:exit();
        }
    }
    public static void dist(int a) throws Exception{
        if(a != 1 && a != 2 && a != 3){
            throw new Exception("输入值错误");
        }
    }
    public static void exit(){
        System.exit(0);
    }

    }

    }
}
