Java输入输出

IO概念

输入输出（I/O）是所有程序都必须的部分——使用输入输出机制，允许程序读取或录入数据，实现数据的交互，Java的IO通过java.io包下的类和接口来支持，在java.io包下主要包括输入、输出两种IO流，每种输入输出流又可分为字节流和字符流两大类。其中字节流以字节为单位处理输入输出操作，而字符流则以字符来处理输入输出操作。

File类

什么是File类？
File类是java.io包下代表与平台无关的文件和目录，也就是说，如果希望在程序中操作文件和目录，都可以通过File类来完成。针对于文件或目录都可以用File来操作，包括新建、删除、重命名。但是File不能访问文件内容本身，如果需要访问文件内容，则需要使用输入输出流。

注:(File类提供了与平台无关的方法来对磁盘上的文件或目录进行操作。因为不同的系统平台对文件路径的描述不尽相同(例如Linux下的根目录和Windows下的各种盘符等)，所以为了做到平台无关，在Java语言中，使用抽象路径等概念。Java自动进行不同系统平台的文件路径描述与抽象文件路径之间的转换。)体现了java的跨平台性

1 访问文件和目录

1.1访问文件名相关的方法
String getName():返回此File对象所表示的文件名或路径名(如果是路径，则返回最后一级子路径名)
String getPath():返回此File对象所对应的路径名
File getAbsoluteFile():返回此File对象的绝对路径
String getAbsolutePath():返回此File对象所对应的绝对路径名
String getParent():返回此File对象所对应目录(最后一级子目录)的父目录名
boolean renameTo(File newName):重命名此File对象所对应的文件或目录，如果成功则返回True，否则false

1.2 文件检测相关的方法

boolean exists():判断File对象所对应的文件或目录是否存在
boolean canWrite():判断File对象所对应的文件或目录是否可写
boolean canRead():判断File对象所对应的文件或目录是否可读
boolean isFile():判断File对象所对应的是否是文件，而不是目录
boolean isDirectory():判断File对象所对应的是否是目录，而不是文件
boolean isAbsolute():判断File对象所对应的文件或目录是否是绝对路径，该方法消除了不同平台的差异，在Linux平台上开始是(/)或者在Windows系统上开头是盘符，则说明它是一个绝对路径

1.3获取常规文件信息

long lastModified():返回文件的最后修改时间
long length():返回文件内容的长度

1.4文件操作相关的方法

boolean createNewFile():当此File对象所对应的文件不存在时，该方法会新建一个文件，如果创建成功则返回true,否则false
boolean delete():删除File对象所对应的文件或路径
void deleteOnExit():注册一个删除钩子，制定当java虚拟机退出时，删除File对象所对应的文件和目录
static File createTempFile(String prefix,String suffix,File directory):在directory目录中创建一个临时空文件，使用给定前缀+系统生成随机数作为文件名

1.5目录操作相关的方法

boolean mkdir():试图创建一个File对象所对应的目录，如果创建成功，则返回true，否则返回false，调用该方法时File对象必须对应一个路径，而不是一个文件
String[] list():列出File对象的所有子文件名和路径名，返回String数组
File[] listFiles():列出File对象的所有子文件和路径，返回File数组
static File[] listRoots():列出系统所有的根路径

注意：Windows路径分隔符使用反斜线()，而JAVA程序中的反斜线表示转义字符，所以如果需要在Windows路径中包括反斜线，应该用两条反斜线，如：E:\aaa\aaa.txt，或者直接使用斜线也可以.Java程序支持将斜线（/）当成平台无关的路径分隔符.
例子：筛选.java文件

import java.io.*;
public class Demo1
{
    public static void main(String[] args){
        File file = new File("E:/java项目");
        findfile(file);
    }
    public static void findfile(File f){
        //运用递归方式排除文件夹
        if(f.isDirectory()){
            File [] ff = f.listFiles();
            for(File x: ff){
                findfile(x);
            }
        }else{
            //剩下的就是文件
            if(f.getName().endsWith(".java")){
                System.out.println(f.getAbsolutePath());
            }
        }
    }
}

java的i/o流（输入输出流）

io流是实现输入输出的基础，可以方便数据操作，在java中把不同的输入输出源(键盘、网络、文件等)抽象为流stream，是从起源source到接收sink的有序数据。

2.1输入输出流
输入流：只能从中读取数据，而不能向其写入数据
输出流：只能写入，不能读取
注意:划分输入输出流是根据内存的角度考虑的

2.2字节流字符流
用法几乎完全一样，只是操作的数据单元不同，字节是8位一字节位处理单元，而字符流则是16位两字节

例子：字符流

import java.util.*;
import java.io.*;
public class Demo
{
    public static Scanner sc = new Scanner(System.in);
    public static StringBuffer sb = new StringBuffer();
    public static void main(String[] args){
        File file = new File("2.properties");
        if(!file.exists()){
            try{
                file.createNewFile();
            }catch(Exception e){
                e.printStackTrace();
            }
        }
        FileReader fr = null;
        try{
            fr = new FileReader(file);
            char[] cc = new char[10];
            int len =0;
            while((len = fr.read(cc)) !=-1){
                sb.append(cc,0,len);
            }
            System.out.println(sb);
        }catch(Exception e){
            e.printStackTrace();
        }


        FileWriter fw = null;
        try{
            fw = new FileWriter(file);
            System.out.println("输入内容");
            sb.append(sc.next()+"\n");
            fw.write(sb.toString());
        }catch(Exception e){
            e.printStackTrace();
        }finally{
            if(fw!=null){
                try{
                    fw.close();
                }catch(Exception e){
                    e.printStackTrace();
                }
            }
        }

        Properties pps = new Properties();
        try{
            FileInputStream fis = new FileInputStream(file);
            pps.load(fis);
        }catch(Exception e){
            e.printStackTrace();
        }
        String s1 = sc.next();
        if(s1.equals(pps.getProperty("username"))){
            System.out.println("yes");
        }

    }
}

例子：字节流

import java.util.*;
import java.io.*;
public class Demo
{
    public static Scanner sc = new Scanner(System.in);
    public static void main(String[] args){
        File file = new File("1.properties");
        if(!file.exists()){
            try{
                file.createNewFile();
            }catch(Exception e){
                e.printStackTrace();
            }
        }
        FileInputStream fis = null;
        FileOutputStream fos = null;
        try{
            fis = new FileInputStream(file);
            byte[] bi = new byte[10];
            int len = 0;
            StringBuffer sb = new StringBuffer();
            while((len = fis.read(bi)) != -1){
                sb.append(new String(bi,0,len));
            }
            System.out.println(sb);
            fis.read(bi);

            fos = new FileOutputStream(file);
            //String so = sb.toString()+sc.next()+'\n';
            //fos.write(so.getBytes());
            String so = new String(sb.append(sc.next()+"\n"));
            fos.write(so.getBytes());

        }catch(Exception e){
            e.printStackTrace();
        }finally{//注意:按道理来说Finally的close应该写在写入流后面就可以了，但是如果写入流在前读取流在后，下面读取流执行的时候会报错"流已经关闭了"，所以应该写在读取流后面，等一切执行完成后再关闭
            if(fos!=null){
                try{
                    fos.close();
                }catch(Exception e){
                    e.printStackTrace();
                }
            }
        }
        Properties pps = new Properties();
        try{
            pps.load(new FileInputStream(file));
        }catch(Exception e){
            e.printStackTrace();
        }
        System.out.println("姓名匹配：");
        String name = sc.next();
        if(name.equals(pps.getProperty("name"))){
            System.out.println("yes");
        }else{
            System.out.println("no");
        }
    }
}
