Java开发模式之MVC

这里首先需要理解几个概念：

 架构：简单的说架构就是一个蓝图，是一种设计方案，将客户的不同需求抽象成为抽象组件，并且能够描述这些抽象组件之间的通信和调用。

 框架：软件框架是项目软件开发过程中提取特定领域软件的共性部分形成的体系结构，不同领域的软件项目有着不同的框架类型。框架不是现成可用的应用系统。而是一个半成品，提供了诸多服务，开发人员进行二次开发，实现具体功能的应用系统。

 设计模式：是一套被反复使用、多数人知晓的、经过分类编目的、代码设计经验的总结，它强调的是一个设计问题的解决方法。

MVC设计模式

 MVC 是一种使用 MVC（Model View Controller 模型-视图-控制器）设计创建 Web 应用程序的模式：
 Model（模型）表示应用程序核心（比如数据库记录列表）,通常模型对象负责在数据库中存取数据。
 View（视图）显示数据（数据库记录）,通常视图是依据模型数据创建的。
 Controller（控制器）处理输入（写入数据库记录）,通常控制器负责从视图读取数据，控制用户输入，并向模型发送数据。

MVC框架

1.在功能划分上面，它强制性的使应用程序输入、处理和输出分开。使用MVC应用程序被分成三个核心部件：模型、视图、控制器。它们各自处理自己的任务。

2.最典型的MVC就是JSP + servlet + javabean的模式。

3.三个核心部件作用：

(1)视图：视图是用户看到并与之交互的界面。视图主要有元素HTML ，Adobe Flash，XHTML，XML/XSL,WML等一些标识语言和Web services。

(2)模型：模型表示企业数据和业务规则。

(3)控制器：控制器接受用户的输入并调用模型和视图去完成用户的需求，所以当单击Web页面中的超链接和发送HTML表单时，控制器本身不输出任何东西和做任何处理。它只是接收请求并决定调用哪个模型构件去处理请求，然后再确定用哪个视图来显示返回的数据。

总结：框架是代码重用，所以我们在使用框架的时候总是引入很多包。
框架是软件，而设计模式是软件的“知识”。所以模式是框架的基础。

在servlet运行过程中MVC所负责内容

1.controller层从HttpRequest当中获取数据
2.封装到一个临时bean当中。
3.传递到service层
4.service层进行业务逻辑，然后向dao层要求数据
5.dao层从数据库提取数据，拿到返回的数据封装到数据模型中，返回到service层
6.controller层将转发数据到View（JSP）
7.由JSP向HttpResponse写入数据
