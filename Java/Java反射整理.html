Java反射整理

 Java程序中的许多对象在运行时都会出现两种类型:编译时类型和运行时类型。当程序在运行时接收到外部传入的一个对象，该对象的编译时类型是Object,但程序又需要调用该对象运行时类型的方法。有时程序需要在运行是发现对象和类的真实信息。第一种情况是编译和运行时都知道类型的具体信息，可以用instanceof进行判断，然后再用强制类型转换符转换。另一种是编译时根本无法预知对象和类的信息，程序只依靠运行时信息来发现，这时必须使用反射。

获得Class对象

1.使用Class类的forName(String clazzName)静态方法。该方法需要传入字符串参数，该字符串参数的值是某个类的权限定类名，要添加全包名。

2.调用某个类的class属性来获取该类对应的Class对象。例如，Person.class将会返回Person类对应的对象。

3.调用某个对象的getClass()方法。该方法是java.lang.Object类中的一个方法，所以所有Java对象都可以调用该方法，该方法会返回该对象所属对应的Class对象。

从Class中获取信息

getConstructors():返回此Class对象对应类的所有构造器。
getMethod().getMethods():返回Class对象的方法。
getField()、getFields():返回Class对象的成员变量。
getDeclaredClasses():返回该class对象对应类里包含的全部内部类。
getDeclaringClass():返回该Class对象对应类所在的外部类。
getInterfaces():返回该Class对象对应类所实现的全部接口。
getSuperclass():返回该Class对象对应类的超类的Class对象。
getPackage():获取此类的包。
getName():返回Class对象所表示的类的名称。

创建对象

1.使用Class对象的newInstance()方法创建实例，要求该Class对象的对应类有默认构造器，执行newInstance()方法时实际上是利用默认构造器来创建该类的实例。

2.使用Class对象获取指定的Constructor对象，再调用Constructor对象的newInstance()方法来创建该Class对应类的实例。

Class<?> clazz = Class.forName(clazzName);//获取对象
clazz.newInstance();//使用默认构造器创建实例

Class<?> jClazz = Class.forName(jclazzName);
Constructor jc = jClazz.getConstructor(String.class);
Object obj = jc.newInstance(属性);

调用方法

Object invoke(Object obj,Object args):obj是执行这个方法的主调，args是执行该方法时传入该方法的实参。

public class Person implements Serializable {

    private String name;
    private int age;
// get/set方法
}
public static void main(String[] args) {
    Person person = new Person("luoxn28", 23);
    Class clazz = person.getClass();

    Field[] fields = clazz.getDeclaredMethods();
    for (Field field : fields) {
        String key = field.getName();
        PropertyDescriptor descriptor = new PropertyDescriptor(key, clazz);
        Method method = descriptor.getReadMethod();
        Object value = method.invoke(person);

        System.out.println(key + ":" + value);
