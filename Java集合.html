Java集合

集合概述

集合是种特别有用的工具类，可用于存储数量不等的对象，并可以实现常用的数据结构队列等。除此之外还可以保存具有映射关系的关联数组。
Java集合大致可分为Set、List、Queue和Map四种体系，其中Queue体系集合是Java5新加的，代表一种队列集合的实现。

集合和数组的区别

1.一旦初始化数组时指定了数组长度，这个数组长度就是不可变的。
2.集合可以保存具有映射关系的数据（也成为关联数组）如成绩表：语文——90、数学——85。
3.数组元素既可以是基本类型的值，也可以是对象，而集合里只能保存对象。（其实保存的是对象的引用变量，但通常习惯上认为集合里保存的是对象）

集合结构

Java集合类主要由两个接口派生而出：Collection和Map,Collection和Map是Java集合框架的根接口，这两个接口又包含了一些子接口或实现类。
Set和List接口是Collection接口派生的两个子接口，分别代表了无序结合和有序集合，Queue是Java提供的队列实现，类似于List。
Map里所有的Map实现类用于保存具有映射关系的数据。

Set集合

set集合通常不能记住元素的添加顺序，不允许包含相同的元素，如果试图把两个相同元素加入同一个Set集合中则会失败，比较常用的是HashSet、TreeSet和EnumSet。

问题：treeset和hashset的区别

相同点：

单列集合，元素不可重复

不同点

1.底层存储的数据结构不同

HashSet底层用的是HashMap哈希表结构存储，而TreeSet底层用的是TreeMap树结构存储

2.存储时保证数据唯一性依据不同

HashSet是通过复写hashCode()方法和equals()方法来保证的，而HashSet通过Compareable接口的compareTo()方法来保证的

3.有序性不一样
HashSet无序，TreeSet有序

List集合

List集合代表一个元素有序、可重复的集合，集合中的每个元素都有其对应的顺序索引。比较常用的如ArrayList和LinkedList。

问题：ArrayList与LinkedList区别

1.linkedList,元素不是连续的，每个元素具有双指针，分别记录前一个元素的地址，和后一个元素的地址
2.ArrayList遍历数元素较快，但是更改数据麻烦，并且线程不安全，LinkedList更改数据快，属于链表结构

import java.util.*;//集合需要导入包
public class Demo
{
    public static void main(String[] args){
        //Collection c = new ArrayList();一般不直接用父接口
        List c = new ArrayList();//声明一个Arraylist集合
        c.add("Taylor");//向集合里添加元素，其实就相当于添加类的对象
        c.add(123);
        System.out.println("c里元素个数"+c.size());//size用来判断集合长度
        System.out.println("第一个元素为"+c.get(1));//get方法是从List里继承的，效果是取第几个元素的值

        //遍历集合方法一
        for(int i=0;i<c.size();i++){//遍历集合方法一:用for循环和get方法遍历并输出每个元素
            System.out.println("元素："+c.get(i));
        }

        //遍历集合方法二
        List<String> d = new ArrayList<String>();//<String>运用泛型，把集合里的值强制转换成String类型
        d.add("Swift");
        //d.add(456);//会报错，找不到合适的方法，因为只能录入String类型的元素了
        d.add("456");
        for(String s:d){//遍历集合方法二:for-each循环有缺陷，只能用同一种类型来表示，所以会用到泛型
            System.out.println("元素："+s);
        }

        /*遍历集合方法二(改)但还是类型转换异常问题
        List f = new ArrayList();
        f.add("Spears");
        f.add(111);
        for(Object o:f){
            String ss = (String)o;
            System.out.println(ss);
        }*/

        //遍历集合方法三
        //注意:在使用迭代的时候不能修改集合元素，否则会报当前修改异常
        Iterator e = c.iterator();//运用迭代器iterator
        while(e.hasNext()){//判断是否有下一个元素，有返回true
            System.out.println("元素："+e.next());//输出下一个元素，必须得一个一个往下读取(hasNext?)，不然只会得到当前一个元素
        }

        //c.clear();//清空集合
        c.remove(0);//删除第一个元素，如果里面没有参数则会删除上一次next()方法返回的值，但后面的元素会往前移
        System.out.println(c.get(0));//第一个元素发生变化

        System.out.println();
        System.out.println();
        System.out.println();

        Set s = new TreeSet();
        s.add(1);
        s.add(2);
        s.add(22);
        //s.add("Binary");这里会出现错误，类型转换异常
        Iterator m = s.iterator();
        while(m.hasNext()){
            System.out.println(m.next());
        }
    }

Map集合

Map用于保存具有映射关系的数据，因此Map集合里保存着两组值，一组值用于保存Map里的key，另外一组值用于保存Map里的value。其中key和value都可以是任何引用类型的数据。Map的key不允许重复，即同一个Map对象的任何两个key通过equals方法比较总是返回false。
HashMap和HashTable都是Map接口的典型实现类。

问题：hashtable hashmap区别

1.hashMap去掉了HashTable 的contains方法，但是加上了containsValue（）和containsKey（）方法。
2.hashTable同步的，而HashMap是非同步的，效率上比hashTable要高,但hashtable安全
3.hashMap允许空键值，而hashTable不允许。
